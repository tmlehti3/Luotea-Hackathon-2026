# Work Orders Dataset — Data Specification

**Dataset:** `work_orders_anonymized.csv`
**Last updated:** 2026-06-03

---

## Overview

This dataset contains **44,265 work orders** from facility management operations at two Valmet industrial sites in Finland. The data spans **May 2017 – June 2026** (~9 years) and covers cleaning services, property maintenance, technical services, and facility services.

The dataset is **anonymized**: all personal names, phone numbers, and email addresses in free-text fields have been replaced with `[NAME]`, `[PHONE]`, and `[EMAIL]` tokens. All other content (technical descriptions, locations, equipment, company names) is preserved.

**Language:** Structured columns are bilingual (Finnish + English). Free-text fields (descriptions, performed actions) are in **Finnish**.

### Work Order Core Fields

| Column | Type | Nullable | Description |
|---|---|---|---|
| `WO_NO` | Integer | No | **Primary key.** Unique ID for an individual work order. |
| `WORK_ORDER_TYPE` | Categorical | No | `EH-työ` (Scheduled work) or `Tilaustyö` (On-demand work). See below. |
| `WORK_ORDER_TYPE_ENG` | Categorical | No | English translation: `Scheduled work` or `On-demand work` |

**Work order types explained:**

- **Scheduled work** (`EH-työ`, 3,241 rows): Generated from the ERP system based on manual programming. Frequencies and scope are based on the contract and service description.
- **On-demand work** (`Tilaustyö`, 41,024 rows): Generated by maintenance personnel, supervisors, customer service, through customer systems via integration, or through forms on the Luotea website. May or may not be included in the contract.

### Free-Text Fields (Anonymized)

| Column | Type | Nullable | Avg length | Description |
|---|---|---|---|---|
| `WORK_DESCRIPTION` | String | Yes | 114 chars | Longer description for the work order. Due to character length constraints on the short description, additional detail may be written here. |
| `WORK_ORDER_DESCRIPTION` | String | No | 16 chars | Shorter description / title for the work order. Max 60 characters (may be truncated). |
| `WORK_ORDER_PERFORMED_ACTION` | String | Yes | 121 chars | Maintenance person's comments when completing the work order. Primarily used for KT (Technical services) and KH (Property maintenance). Cleaning services (SP) do not have the same requirement. |

These three columns contain the anonymized content. Common patterns in `WORK_ORDER_PERFORMED_ACTION`:

- `Työn Tilaaja: [NAME]` — work orderer
- `Tilattu työ: ...` — ordered work description
- `Tehty työ: ...` — completed work description
- `Ilmoittaja: [NAME] [PHONE]` — reporter with contact info
- Date-prefixed entries: `DD.MM.YYYY - action taken`

|---|---|---|

Anonymization was applied **only** to the three free-text columns: `WORK_DESCRIPTION`, `WORK_ORDER_DESCRIPTION`, and `WORK_ORDER_PERFORMED_ACTION`. All other columns are unmodified.

## Notes for Participants

1. **NULL values** appear as the literal string `NULL` in some columns (not empty strings).
2. **Free-text is in Finnish.** Use translation or Finnish NLP tools if needed for text analysis.
3. **WORK_ORDER_DESCRIPTION** is a short title (max 60 chars, often truncated). For full context, combine with `WORK_DESCRIPTION` and `WORK_ORDER_PERFORMED_ACTION`.
4. **SLA violation rate is high (53.8%)** — this reflects the data as-is and may be a useful analysis target.
5. **WORK_FINISHED_DAYS** can be very large (up to 1,261 days) for long-running or re-opened work orders.
6. **Encoding:** UTF-8 with BOM. Most tools handle this automatically; if the first column name appears garbled, strip the BOM.
7. **ASSIGNMENT_TYPE and PRIORITY_ID** do not apply to cleaning services (SP) and are NULL for those rows.
8. **PM_NO** can be used to join this dataset with the Scheduled Maintenance dataset.
